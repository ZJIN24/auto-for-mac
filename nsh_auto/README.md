# MacClickStudio

一个用 SwiftUI 写的 macOS 自动化工作台，目标是把“抓抓 + 脚本编写 + 窗口锁定 + OCR”合成一个更接近按键精灵 / 抓抓思路的原型。

## 当前能力

### 多工作区

- 启动台：统一打开全景工作台 / 抓抓 / 窗口实验室 / 脚本中心
- 菜单栏面板：可直接截图、按锁定窗口截图、打开/保存项目、运行脚本、打开各子程序
- 命令菜单：支持用快捷键快速打开各工作区，也支持项目级新建 / 打开 / 保存

### 抓抓能力

- 截取主屏画面并在画布上直接抓点
- 支持对当前锁定窗口单独截图，做窗口内找点、找色、找图更方便
- 实时显示鼠标悬停坐标、偏移后的取色坐标和颜色值
- 支持右键菜单：复制坐标、复制颜色、加入比色点、添加点击步骤、按点位锁定窗口
- 支持区域拉框，显示左上 / 右下坐标
- 支持偏移抓点，避开鼠标指针遮挡

### 图像 / 颜色识别

- 保存模板图并设置推荐相似度
- 支持模板预处理：原图 / 灰度 / 高对比
- 支持区域找图点击
- 支持多点比色等待
- 支持区域找色点击

### 窗口 / 句柄 / 控件

- 枚举当前可见窗口，显示窗口标题、PID、窗口 ID（把它当作 macOS 侧的“句柄近似”）
- 通过当前选中点反查并锁定目标窗口
- 支持显示相对于窗口的坐标
- 支持读取点位下无障碍控件信息：role / subrole / title / value / identifier
- 支持两种事件投递模式：
  - 系统前台投递
  - 目标进程投递（best-effort，更适合你要的“只对某个程序操作”）

### 脚本能力

- 可视化步骤脚本：等待 / 点击 / 多点比色 / 找图点击 / 找色点击
- 默认提供 Python 风格脚本工作台，也保留 JavaScript 模式
- Python 风格脚本支持 `def`、`if / elif / else`、`while`、`for range`、`return`、`break`、`continue`
- 内置 helper：`click`、`click_relative`、`get_color`、`get_color_relative`、`wait_color`、`wait_color_relative`、`find_image`、`find_image_in_window`、`find_color`、`find_color_in_window`、`ocr_text`、`ocr_window`、`current_window`
- 内置可搜索函数库：查看签名、关键字、示例，并可一键插入 / 应用示例 / 复制示例
- 可把当前点位 / 颜色 / 选区 / 窗口信息插入脚本编辑器
- 支持脚本新建 / 打开 / 保存 / 另存为，也支持项目级新建 / 打开 / 保存 / 另存为

### OCR

- 对当前选区做中英文 OCR
- 可复制 OCR 结果，也可在脚本中调用 OCR

## 运行方式

```bash
swift run
```

也可以直接用 Xcode 打开 `Package.swift`。

## 首次运行要做的事

应用需要两个权限：

- 屏幕录制：用于截图、取色、找图、OCR
- 辅助功能：用于发送鼠标事件、控件探测

在 UI 里先点“申请权限”，然后再点“截图”。

注意：这个项目当前是 Swift Package 可执行程序，不是传统 `.app` 工程。

- 如果你是在 Xcode 里直接运行，macOS 往往会把权限记到 `Xcode` 这个宿主上
- 如果你是用 `swift run` 或终端启动，权限通常记到 `Terminal` / `iTerm`
- 屏幕录制和辅助功能在勾选后，常常需要**完全退出并重新打开**当前宿主程序才会生效

## 推荐工作流

你也可以按子程序拆开用：

- `启动台`：进应用后的总入口
- `抓抓`：专心做找点、取色、模板和 OCR
- `窗口实验室`：专心锁窗口、看 PID、看窗口 ID、研究控件
- `脚本中心`：专心编步骤和 JavaScript
- `全景工作台`：需要全链路联调时再打开


1. 点“申请权限”
2. 点“截图”拿到当前画面
3. 用“点位模式”抓点击坐标、颜色坐标
4. 需要时设置偏移抓点
5. 在目标程序上取点，然后点“按点位锁定窗口”
6. 需要后台/定向操作时，把事件投递切到“目标进程投递”
7. 用“区域模式”拉框做模板图或 OCR 区域
8. 通过步骤脚本或 Python 风格 / JavaScript 脚本编排逻辑

## 关于“只在一个窗口上操作”

这版已经做了两层支持：

- **窗口锁定 + 相对坐标**：窗口移动后，点击步骤仍可根据窗口位置重新计算目标点
- **按目标 PID 投递事件**：点击事件可以不走全局 HID，而是直接投递给目标进程

但要说明一个 macOS 边界：

- 目标进程是否接受后台鼠标事件，取决于具体程序实现
- 常规 App 通常更容易成功
- 一些游戏、沙盒 App、Metal / 自绘界面可能会忽略后台事件或要求窗口前置

所以这里的后台窗口点击是 **best-effort**，不是所有程序都 100% 通用。

## 目前边界

- 目前以主屏为主，不做多显示器拼接坐标
- 还不是完整的节点式流程图编辑器
- 已支持项目级与脚本级文件保存，但还没做真正的文档型多窗口编辑
- 没接断点调试、脚本变量观察器、模板库云同步

## 后续建议

如果你要继续往“按键精灵 for mac”方向做，建议下一步补：

- 多窗口 / 多子程序工作区
- 节点流程图编辑器
- 变量、条件分支、循环面板化
- 项目保存 / 导入 / 模板库
- 窗口内控件树浏览器
- 更强的后台事件策略与兼容性适配
